The Typography Bandit

Web typography best practices and resources for type criminals.

About

We are the Typography Bandit, design vigilantes protecting the internet from bad type choices. We are legion. Before you commit another type crime, please review the following typo­graphy best practices and resources so together we can make the web a safer place for type to live.

Best Practices

  • Make your body text readable, i.e. at least 19px for desktop screens and never under 16px for mobile screens. Setting html: {font-size: 16px;} and then assigning rem values to your headings and body text is one way to do this.
  • For fluid font sizing without media queries, consider using the CSS clamp() function. A range like clamp(1rem, 1rem + .5vw, 1.7rem) should work well for your body text. The first value is the minimum font-size (for mobile screens), the middle value is the target size, and the final value is the maximum size. Or, to cover both modern and legacy browsers, simply use the calc() function, i.e. calc(1rem + .5vw).
  • Leave body text letter spacing alone, i.e. letter-spacing: 0px. Subheadings in all caps may require a little extra breathing room.
  • Set your maximum line width to no more than 70 characters. That includes spaces. The best way to do this is to assign ch unit values to your text containers, i.e. .text p {max-width: 60ch;}.
  • Choose background, foreground (text), and accent colors with high contrast for maximum accessibility. Use the WebAIM Contrast Checker (link below) to see if you're in the ballpark.
  • Never justify text, and try to avoid centering body text.
  • Use no more than three, and preferrably two, typefaces on one website. It's not just a question of aesthetics. Loading all those font files will slow down your website and generate more CO2.
  • Use monospaced fonts sparingly. They're harder on the eyes.
  • Consider system fonts for quicker page loads. Some of them are pretty good. See our link to Modern Font Stacks below.
  • Host your own fonts rather than linking to a CDN. They generally load faster this way and so help with FOIT (Flash of Invsible Text) and FOUT (Flash of Unstyled Text). Just remember to preload your fonts in the head.

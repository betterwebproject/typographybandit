About

We are the Typography Bandit, design vigilantes protecting the internet from bad type choices. We are legion. Before you commit another type crime, please review the following free typo­graphy best practices and resources so together we can make the web a safer place for type to live.

Best Practices

  • Leave body text letter spacing alone, i.e. letter-spacing: 0px.
  • Make body text font size at least 17px for desktop screens and never under 16px for mobile screens. Ideally, you should set your html to 16px and then assign rem values to your headings and body text. (The body text on this site is set at 1.25rem, or 20px.)
  • Set your maximum line width to no more than 75 characters. That includes spaces. You can assign rem values to your text containers to do this. The text on this site is set at max-width: 40rem, for example. Based on a root font size of 16px, we get a line width of around 640px. That works out to roughly 72 characters per line with our body text at 20px.
  • Never justify text, and try to avoid centering body text.
  • If you have the urge to use marquee text, visit a site that uses it. Look at it for a minute and think again.
  • Choose background and foreground (text) colors with high contrast. Use the WebAIM Contrast Checker (link below) to see if you’re in the ballpark.
  • Use monospace fonts sparingly.
  • Use no more than three, and preferrably two, typefaces on one website. It’s not just a question of aesthetics. Loading all those font files will slow down your website and generate more CO2.
  • If you’ve got fast servers, and are using the @font-face CSS at-rule, consider using “font-display: optional” instead of “font-display: swap” to avoid FOIT (flash of invisible text) and FOUT (flash of unstyled text), and layout shifts. Just remember to preload your fonts in the head and to use a similar fallback font.
